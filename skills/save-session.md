# /save-session — Save state before starting a new session

Write a detailed state file to `.context-guard/state.md` in the current project directory. This file must allow a fresh Claude session to pick up exactly where we left off.

## Steps

1. Create `.context-guard/` directory in the project root if it doesn't exist
2. Write `.context-guard/state.md` with this structure:

```markdown
# Session State — [current date/time]

**Session ID:** `[current session ID]`
**Project:** [project directory]
**Resume:** `claude -r [session ID]`

## What Was Done
[Detailed summary of everything accomplished in this session — features built, bugs fixed, decisions made, problems encountered and solved]

## Current Status
[Where things stand right now — what's working, what's not, any blockers]

## Files Modified
[List of key files created or modified with brief description of changes]

## Next Steps
[Exact next actions to take, in order, to continue the work]

---
_Generated by claude-context-guard_
```

3. Print the session ID and the path to the state file
4. Tell the user: "Start a new session with `claude -r [session ID]` or open a fresh session and say: Read `.context-guard/state.md` to continue."
